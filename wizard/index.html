Where will the race data live?

The backend stores all your cars, heats, and results. Pick based on your setup.

Recommended

☁️ Supabase

  • Free tier available
  • No local server needed
  • Works anywhere with internet
  • Email + file storage included
  • Requires internet connection
  • Needs a free Supabase account
Local

🖥️ Local (SQLite + Python)

  • Fully offline — no internet needed
  • Runs on Raspberry Pi or any PC
  • No account required
  • GPIO sensor support
  • Python must be running
  • File storage is local only

Use 0.0.0.0 to accept connections from other devices on your network

Where will the race pages be hosted?

The frontend is the web interface parents, officials, and display screens use.

⚠️ HTTPS Warning
GitHub Pages forces HTTPS. A local backend (on your PC or Pi) runs plain HTTP. Browsers block HTTP requests from HTTPS pages as "mixed content."

To use this combination you need: your own domain with HTTPS on the local server (e.g. via Cloudflare Tunnel), or enable Chrome's experimental "Insecure origins treated as secure" flag.

Recommended: Switch to Local frontend when using a local backend.

How will race results be recorded?

Choose how finish order is captured — manually by officials, or automatically by sensors.

Manual

📋 Heat / Round System

  • No extra hardware needed
  • Officials tap finish order on screen
  • Works everywhere
External sensor

📡 Sensor Bridge (ESP32)

  • Automatic timing
  • ESP32 POSTs results over WiFi
  • Works with any backend
  • Requires ESP32 + IR sensors
GPIO / Pi only

🔌 Sensor Bridge (GPIO, same device)

  • No second device needed
  • IR sensors wired directly to Pi
  • Lowest latency
  • Requires Raspberry Pi or Linux with GPIO
  • Local backend only

Wired to your starting gate switch. Arms the race timer on gate open. Leave blank to arm via the gate broadcaster ESP32 or manually.

Network setup

Optionally make this device broadcast its own WiFi network so race devices connect directly.

ℹ️ Hotspot is only available when using a local backend. Switch to local backend in step 1 to enable this option.

📶 No hotspot (use existing WiFi)

Devices connect to the existing WiFi and access the race server by IP address

Hotspot

📡 Regular hotspot

Pi broadcasts a WiFi network. Devices connect to it, then navigate to the Pi's IP address, or any URL if using captive portal mode (e.g. derby.com).

Captive portal

🌐 Captive portal

Pi broadcasts WiFi. Any URL a device visits automatically redirects to the race app — like hotel WiFi. Requires root (sudo python3 run.py).
